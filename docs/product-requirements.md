# 小说转剧本后端 — 项目需求文档

## 1. 项目背景

小说作者在作品完成后，常有将其改编为剧本的需求（投递影视公司、参加创投、或个人练习）。但小说到剧本的转换涉及叙事结构重组、场景拆分、对白提炼等繁琐工作，手动完成成本极高。

现有 AI 工具多输出纯文本剧本，缺乏结构化编辑能力，作者无法在生成的剧本上进行精细化修改。市场缺少一款面向小说作者、输出结构化剧本、支持迭代打磨的工具。

## 2. 产品目标

构建一个 AI 辅助的剧本创作后端服务，让小说作者能够：

- 上传自己的小说文本（≥3 章）
- 通过多 AI 智能体协作，自动生成结构化剧本
- 以 YAML 格式获取、查看和编辑剧本
- 支持多轮对话式迭代修改

## 3. 目标用户

| 用户类型 | 描述 | 核心需求 |
|---|---|---|
| 独立小说作者 | 有完结/连载中小说，希望尝试剧本改编 | 快速获得剧本初稿，降低入门门槛 |
| 编剧/写作爱好者 | 有小说创作经验，想学习剧本写作 | 通过 AI 生成的范例学习剧本结构 |
| 影视工作室 | 批量评估小说 IP 的影视化潜力 | 快速将多部小说转为剧本样稿用于评估 |

## 4. 功能需求

### 4.1 用户账号与安全管理

| ID | 功能 | 描述 |
|---|---|---|
| F-01 | 用户注册/登录 | 手机号注册，JWT 认证 |
| F-02 | 自定义 AI 模型 | 用户可配置自己的 API Key、模型、参数，支持按场景绑定不同模型 |
| F-03 | API Key 安全存储 | 用户的 API Key 加密存储，不暴露给前端 |

### 4.2 小说项目管理

| ID | 功能 | 描述 |
|---|---|---|
| F-04 | 项目 CRUD | 创建、查看、编辑、删除小说项目 |
| F-05 | 章节管理 | 上传小说的各个章节文本，支持 ≥3 章 |
| F-06 | 素材管理 | AI 自动从小说中提取角色等素材，用户可查看和编辑 |

### 4.3 剧本智能生成（核心）

| ID | 功能 | 描述 |
|---|---|---|
| F-07 | 故事线分析 | AI 分析全部章节，提取核心故事线、情节结构、角色关系 |
| F-08 | 分集规划 | 根据故事线自动规划分集（集数、每集覆盖的章节范围） |
| F-09 | 逐集大纲生成 | 按规划依次生成每集的详细大纲 |
| F-10 | 质量审核 | 导演 Agent 审核每集大纲和剧本质量，不通过则退回修改 |
| F-11 | 续写支持 | 长篇小说自动分 chunk 多次调用 AI，支持续写 |
| F-12 | 用户交互 | 用户可在生成过程中通过对话对 Agent 提出修改意见 |

### 4.4 剧本管理

| ID | 功能 | 描述 |
|---|---|---|
| F-13 | 剧本流式生成 | 生成过程 SSE 流式返回，前端可实时展示进度 |
| F-14 | 剧本保存/编辑 | 用户可手动编辑 YAML 格式剧本 |
| F-15 | 剧本导出 | 支持单集 YAML 导出和全集批量导出 |

### 4.5 内容安全

| ID | 功能 | 描述 |
|---|---|---|
| F-16 | Prompt 指纹检测 | 检测注入攻击 |
| F-17 | 品牌名替换 | 自动替换输出中的品牌名 |
| F-18 | 流式内容守卫 | 实时拦截不安全内容 |

## 5. 用户工作流

```
① 注册/登录
   ↓
② 创建小说项目
   ↓
③ 上传 ≥3 章小说文本
   ↓
④ 发起剧本生成（触发多智能体管线）
   ├── AI 故事师分析章节 → 生成故事线
   ├── AI 规划师制定分集规划
   ├── AI 大纲师生成逐集大纲
   └── AI 导演审核质量 → 通过/退回
   ↓
⑤ 获得 YAML 剧本初稿
   ↓
⑥ 查看/编辑/导出剧本
   └── 可返回步骤 ④ 重新生成或对话修改
```

## 6. 非功能需求

| 类别 | 要求 |
|---|---|
| **性能** | 剧本生成支持流式返回，首块响应时间 ≤ 5 秒（依赖 AI 模型响应速度） |
| **兼容性** | 输出标准 YAML 格式，前端可直接解析渲染；兼容 OpenAI 兼容 API |
| **安全性** | API Key 加密存储（AES-256-GCM）；JWT 认证；Prompt 注入防护 |
| **可扩展性** | 模型配置支持按场景绑定，用户可自由切换不同 AI 模型 |
| **数据完整性** | 剧本数据 round-trip 一致（对象 → YAML → 对象） |

## 7. 局限性（当前版本不包含）

- **不提供前端 UI**：当前版本仅提供后端 API 和 SSE 流式接口
- **不支持 <3 章**：管线要求至少 3 章文本以保证分析质量
- **不提供多语言支持**：目标用户为中文小说作者，剧本输出为中文
- **不直接支持非 OpenAI 兼容 API**：但用户可通过自定义模型配置接入兼容接口
